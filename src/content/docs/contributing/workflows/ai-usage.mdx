---
title: AI Usage Policy
description: How AI assistants can be used to improve contributions
---

Any contributor is required to fully understand a piece of work that he proposes. No fully automated AI-generated work is allowed. Please do not make such contributions, they will be rejected.

## Recommended Scenarios

### Code

- Discuss architecture proposals and APIs with AI
- Review production code with AI, but apply comments manually

### Tests

- Generate unit tests, then do manual code review
- Generate test and experiment harnesses with manual code review

### Docs

- Review documentation with AI, but apply comments manually

From any AI-assisted texts we explicitly require no emoji and reduced usage of bullet lists and emphasis.

Avoid using AI to write documentation entirely as it becomes too vague. Preserve your individual style of writing and feel proud of it, even if it isn't as smooth as AI-generated one.
